v1.0.4 - Januari 20 2014

Fixed the hover effect for the portfolio thumbnails

Because of the previous 'equal height' update, the hover effect for the portfolio thumbnails stopped working. To get them to work again, here's what you can do if you bought Magnum before:

1. In the HTML file, around the portfolio thumbnails section, add the following classname: fade-it. The code would be as follows:


<!-- Portfolio thumb 1 -->
<div class="four columns medium-six columns small-twelve columns fade-it">
    <div class="portfolio-thumb">
        et cetera

2. In the main.js file around line 149, add this fade-it class as well in the code, like the example below:


// 05. Fading divs - Fade a div except the one that's hovered
if($('html').hasClass('no-touch')){
    $('.fade-it, .image-thumb, .service-item').hover(function(){
        $(this).siblings().addClass('fade');
    }, function(){
        $(this).siblings().removeClass('fade');
    });
}

Note that the portfolio-thumb class is removed within this new js code. That's how it should be.
That's it!

v1.0.3 - Januari 13 2014

Added the 'equal' class for equal height columns

You can use the equal class for - for example - the portfolio thumbnails if the total height of these portfolio items aren't equal in height.

Old situation:

New situation:

For anyone who has download Magnum before, here's what you can do:
1. At the bottom of the HTML file, where you'll find the javascripts, add the following line after the bxslider or stickyscroll line:

<script src="js/lib/jquery.equal.js"></script>

2. Don't forget to copy this file from the new files and paste it into your js/lib folder.
3. Open the main.js file from the js folder and scroll down to the bottom of the file. Add the following lines:


/* 09. Use the .equal class on a row if you want the columns to be equal in height */
$(document).ready(function(){
	$('.equal').children('.columns').equalizeHeight();
	$(window).resize(function(){
			$('.equal').children('.columns').equalizeHeight();
	})
})

4. Open the style.css file and hit 'ctrl + f' or 'cmnd + f' and search 'portfolio-thumb em'. Add the following lines after this CSS class:

				
.portfolio-thumb b,
.portfolio-thumb em {
	padding-left: 25px;
	padding-right: 25px;
}

5. In the HTML document, where you'll find the portfolio thumbnails, add the equal class after the row margin classes:

					
<!-- Thumbnails -->
<div class="row margin equal">

6. If you want to prevent all the white blocks to become as tall as the item with the most content in it, wrap the portfolio-thumb class inside a new div:


<!-- Portfolio thumb -->
<div class="four columns medium-six columns small-twelve columns portfolio-thumb">
	<a href="project.html">
		<img src="images/portfolio-thumbnail.jpg" alt="portfolio-thumbnail">
		<i class="fa fa-trophy"></i>
		<b>Project Title</b>
		<em>Project Page</em>
	</a>
</div>

Becomes:


<!-- Portfolio thumb -->
<div class="four columns medium-six columns small-twelve columns">
	<div class="portfolio-thumb">
		<a href="project.html">
			<img src="images/portfolio-thumbnail.jpg" alt="portfolio-thumbnail">
			<i class="fa fa-trophy"></i>
			<b>Project Title</b>
			<em>Project Page</em>
		</a>
	</div>
</div>

That's it!

v1.0.2 - Januari 8 2014

Fixed a bug in the sticky menu that was visible while scrolling down the page

For anyone who has download Magnum before, here's what you can do:
1. At the bottom of the HTML file, where you'll find the javascripts, add the following line after the bxslider line:

<script src="js/lib/jquery.sticky.js"></script>

So this would be as following:


<!-- Javascript -->
<script src="js/lib/jquery.min.js"></script>
<script src="js/lib/jquery.fittext.js"></script>
<script src="js/lib/jquery.magnific-popup.min.js"></script>
<script src="js/lib/jquery.bxslider.min.js"></script>
<script src="js/lib/jquery.sticky.js"></script>

2. Open the main.js file from the js folder and scroll down to the bottom of the file. Delete the following lines:


// 08. Fixed menu - Fix the menu to the top after you scroll past it
var fixedmenuTop = $('.fixedmenu').offset().top;
$(window).scroll(function(){
	var currentScroll = $(window).scrollTop();
	if (currentScroll >= fixedmenuTop) {
		$('.fixedmenu').css({
			position: 'fixed',
			top: '0',
			left: '0'
		});
	} else {
		$('.fixedmenu').css({
			position: 'relative'
		});
	}
});

3. Replace these lines with the following code:


// 08. Fixed menu - Fix the menu to the top after you scroll past it
$(window).load(function(){
	'use strict';
	$(".fixedmenu").sticky({ topSpacing: 0 });
});

4. Open the style.css file and hit 'ctrl + f' or 'cmnd + f' and search 'menu-wrap'. (It's probably the second result.) You'll see the following lines:


/* ******************** Responsive settings for the menu ******************** */
@media only screen and (max-width: 768px){

	.menu-wrap {
		width: 100%;
	}

5. Add the following lines above it:


/* ******************** Responsive settings for the menu ******************** */
@media only screen and (max-width: 768px){

	.sticky-wrapper,
	.main-menu {
		height: 50px!important;
	}
	
	.menu-wrap {
		width: 100%;
	}

That's it!

Contact form error message bug fix

Fixed a bug from the contact form's error message that was visible on small screens. For anyone who has download Magnum before, here's what you can do:
1. In the HTML file, add an extra row after the message div in the contact section:

Old version:


<div class="row">

	<!-- Succes message after sending the form, see also the php file around line 90 -->
	<div class="twelve columns">
		<div id="message"></div>
	</div>

	<!-- The contactform -->
	<form method="post" action="contact.php" name="contactform" id="contactform">

Updated version:


<div class="row">

	<!-- Succes message after sending the form, see also the php file around line 90 -->
	<div class="twelve columns">
		<div id="message"></div>
	</div>
	
</div>
<div class="row">

	<!-- The contactform -->
	<form method="post" action="contact.php" name="contactform" id="contactform">

2. In the style.css file, hit 'ctrl + f' or 'cmnd + f' and look for error_message. You can copy and paste the new version and overwrite the old version.

Old version:


.error_message {
	display: block;
	height: 40px;
	line-height: 40px;
	background: #fbe3e4;
	color: #8a1f11;
	border: 1px solid #fbc2c4;
}

Updated version:


.error_message {
	display: block;
	padding: 5px;
	background: #fbe3e4;
	color: #8a1f11;
	border: 1px solid #fbc2c4;
}

That's it!
